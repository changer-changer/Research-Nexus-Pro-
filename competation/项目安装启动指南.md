# Research-Nexus Pro — 项目安装与启动指南

> 本文档面向比赛答辩现场演示与日常开发调试，涵盖完整的环境准备、依赖安装、前后端启动流程。

---

## 一、环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.9+ | 后端运行环境 |
| Node.js | 18+ | 前端运行环境 |
| npm | 9+ | 包管理器（随 Node.js 安装） |
| 操作系统 | Linux / macOS / Windows (WSL) | 推荐 Ubuntu 22.04+ |

### 1.1 快速检查环境

```bash
# 检查 Python 版本
python3 --version

# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version
```

如果任一命令不存在或版本过低，请先安装/升级对应环境。

## 三、首次安装（只需执行一次）

### 3.1 克隆/进入项目目录

```bash
cd /path/to/research-nexus-pro
```

### 3.2 安装前端依赖

```bash
npm install
```

> 这会安装 `package.json` 中定义的全部前端依赖（React 18、ReactFlow、D3、Framer Motion、Zustand、Tailwind CSS 等）。

### 3.3 安装后端依赖

```bash
cd backend

# 创建 Python 虚拟环境
python3 -m venv venv

# 激活虚拟环境 (Linux/macOS)
source venv/bin/activate

# 激活虚拟环境 (Windows CMD)
# venv\Scripts\activate.bat

# 激活虚拟环境 (Windows PowerShell)
# venv\Scripts\Activate.ps1

# 安装 Python 依赖
pip install -r requirements.txt
```

> 依赖包括：FastAPI、Uvicorn、Pydantic、NetworkX、NumPy、pytest、OpenAI SDK 等。

### 3.4 检查必需数据文件

```bash
cd ..  # 回到项目根目录

# 确认 EXTRACTED_DATA.json 存在
ls EXTRACTED_DATA.json
```

如果该文件不存在，系统将无法正常运行。请确保它位于项目根目录下。

## 四、日常启动（开发调试）

### 方式一：一键启动（推荐）

项目根目录下执行：

```bash
./start-research-nexus.sh
```

脚本会自动完成以下工作：
1. 检查 Python / Node.js 环境
2. 确认 `EXTRACTED_DATA.json` 存在
3. 创建/激活 Python 虚拟环境
4. 安装缺失的依赖
5. 启动后端服务（端口 8000，后台运行）
6. 启动前端 Dev Server（端口 5173，后台运行）
7. 输出访问地址

**输出示例：**
```
========================================
  Research-Nexus Pro 启动成功!
========================================

  前端: http://localhost:5173
  后端: http://localhost:8000
  API文档: http://localhost:8000/docs

  按 Ctrl+C 停止服务
```

> 按 `Ctrl+C` 即可同时停止前后端服务。

### 方式二：手动分步启动（调试排错时用）

**终端 1 — 启动后端：**

```bash
cd backend
source venv/bin/activate
python -m app.api.main_local
```

> 后端将启动在 `http://localhost:8000`
> API 文档：`http://localhost:8000/docs`
> 健康检查：`http://localhost:8000/api/health`

**终端 2 — 启动前端：**

```bash
# 在项目根目录（另一个终端窗口）
npm run dev
```

> 前端将启动在 `http://localhost:5173`
> Vite 会自动代理 `/api` 请求到 `http://localhost:8000`

### 方式三：构建生产版本

```bash
# 构建前端静态文件
npm run build

# 预览生产构建
npm run preview
```

> 生产构建输出在 `dist/` 目录，当前项目配置为 GitHub Pages 部署。

## 六、环境变量（可选）

## 七、常见问题排查

### Q1：`start-research-nexus.sh` 提示权限不足

```bash
chmod +x start-research-nexus.sh
```

### Q2：后端启动时报 `ModuleNotFoundError`

```bash
cd backend
source venv/bin/activate
pip install -r requirements.txt
```

确保虚拟环境已激活，且依赖已完整安装。

### Q3：前端页面空白，控制台报错

```bash
# 清除缓存重新安装
rm -rf node_modules
npm install
npm run dev
```

### Q4：前后端端口被占用

```bash
# 查看占用端口的进程
lsof -i :8000   # 后端端口
lsof -i :5173   # 前端端口

# 结束对应进程
kill -9 <PID>
```

### Q5：CORS 跨域错误

Vite Dev Server 已配置代理规则，正常情况下不会出现。如果手动修改了端口，请检查 `vite.config.ts` 中的 `server.proxy` 配置。
