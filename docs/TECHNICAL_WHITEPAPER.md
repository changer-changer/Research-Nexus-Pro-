## 3. Phase 2: Agent 集群分析（核心）

### 3.1 SubAgent 任务设计

**任务 Prompt 结构**

```
【系统角色】
你是专业的学术论文分析专家，拥有博士学位。

【核心约束】
1. 你必须完整阅读PDF的每一个章节
2. 你必须提取真实的表格数据，不能编造
3. 所有引用必须能在原文中找到对应位置
4. 如果信息不确定，明确标注"未在原文中找到"
5. 不准偷懒，不准省略细节

【输入信息】
- PDF路径: {pdf_path}
- 论文标题: {title}
- ArXiv ID: {arxiv_id}

【工具使用】
使用 paper-reader 提取PDF内容：
python3 read_paper.py {pdf_path} --full

【输出要求】
生成6部分详细报告，保存到 {output_path}

Part 1: 研究背景
- 领域背景
- 前人研究基础（引用3-5篇关键论文）
- 已有实验基础

Part 2: 研究问题
- 解决的具体问题
- 现有方法局限性（引用原文）
- 核心假设

Part 3: 核心创新工作
- 方法/系统架构详细描述
- 技术细节（网络结构、公式）
- 与前人方法对比表格

Part 4: 关键实验设计
- 数据集（名称、规模）
- Baseline方法
- 实验结果表格（真实数据）
- 消融实验

Part 5: 核心洞察与结论
- 主要发现
- 领域洞察
- 实践建议

Part 6: 待解决问题与未来方向
- 本文局限性
- 待解决问题
- 未来研究方向

【质量自检清单】
分析完成后，检查：
□ 我是否读完了Introduction的每一句?
□ 我是否理解了Method的每个公式?
□ 我是否提取了所有实验表格?
□ 我是否找到了Related Work的关键引用?
□ 我的引用是否都能在原文中找到?
□ 我是否标注了不确定的信息?

【格式要求】
- Markdown格式
- 长度不少于3000字
- 包含至少3个数据表格
- 至少10处原文引用
```

## 6. 关键技术难点

### 6.1 PDF解析的准确性

**挑战**：学术论文PDF格式复杂，表格、公式、图表难以准确提取

**解决方案**：
- 使用 `pdfplumber` + `pymupdf` 双引擎
- 对图表使用OCR识别
- 人工校验关键数据

### 6.2 Agent上下文限制

**挑战**：单Agent无法处理超长PDF

**解决方案**：
- Multi-Agent架构，每篇论文独立Agent
- PDF分段读取，Agent专注于单篇论文
- 报告结构化，便于后续整合

### 6.3 防止Agent幻觉

**挑战**：AI容易编造数据、虚构引用

**解决方案**：
- Prompt工程：明确要求标注来源
- 自检机制：要求Agent完成检查清单
- 引用验证：格式检查 + 可追溯性
- 开源透明：允许人工抽查验证
