# Gentle Manipulation Policy Learning via Demonstrations from VLM Planned Atomic Skills

**arXiv ID**: 2511.05855
**别名**: Visual-Tactile Diffusion Policy (VTDP/VT-DP)

## 1. 研究背景

**领域现状**: 机器人操作正从简单的拾取-放置向长序列、接触丰富（Contact-Rich）的任务演进。这类任务（如擦拭、精细装配）要求机器人由于长时间与环境交互，必须具备“轻柔（Gentle）”的操作能力以避免损坏物体。
**前人研究**: 传统的模仿学习（Imitation Learning）依赖大量专家数据，采集成本高昂且难以保证数据的一致性。现有的VLM（视觉语言模型）规划方法虽然能进行其高层任务分解，但往往忽略了底层的物理约束（如接触力），导致生成的动作在物理世界中不可行或具有破坏性。
**应用痛点**:

1. **数据稀缺**: 长序列柔性操作的高质量示教数据极难获取。
2. **力感知缺失**: 纯视觉方案无法感知接触力，难以实现精细力控。
3. **VLM幻觉**: VLM直接生成底层控制指令容易产生幻觉，导致执行失败。

## 2. 研究问题

**核心问题**: 如何在缺乏大规模人类示教数据的情况下，利用VLM的规划能力生成包含力信息的长序列演示数据，并训练出具备力感知能力的轻柔操作策略？
**拟解决难题**:

- 将高层语义规划转化为底层的、符合物理约束（力控）的动作序列。
- 实现视觉与触觉的有效融合，以支持高精度的接触操作。

## 3. 核心创新工作

**系统/框架**: 提出 **VASK** (VLM Planned Atomic Skills) 框架。

1. **原子技能库 (Atomic Skills)**: 预定义并训练了一组具备力感知能力的元技能（如`MoveTo`, `Grasp`, `Press`, `Wipe`）。这些技能在仿真中通过强化学习（RL）训练，显式引入了力约束。
2. **VLM任务分解**: 利用VLM将高层语言指令分解为原子技能序列。
3. **数据生成**: 在仿真环境中执行VLM规划的技能序列，生成包含视觉、触觉和本体感觉的完整演示数据。

**算法设计**:

- **VT-DP (Visual-Tactile Diffusion Policy)**: 提出一种视触觉扩散策略，用于从生成的数据中蒸馏全任务策略。
  - **输入**: 视觉点云（256点） + 触觉点云（128点）。
  - **结构**: 独立的视觉和触觉编码器，融合后输入Diffusion Head。
  - **优势**: 相比纯RGB或单模态点云，VT-DP能更有效地捕捉接触几何特征。

## 4. 关键实验设计

**实验平台**:

- **仿真**: 包含力学模拟的物理引擎（具体未详述，可能是Isaac Gym或PyBullet），使用TacSL软接触模型模拟触觉及其变形。
- **真机**: 真实机械臂 + Realsense D415（视觉） + GelSight Mini（触觉）。使用数字孪生（Digital Twin）技术将真机点云对齐到仿真空间。

**数据集/任务**:

- **长序列任务**: 开瓶盖（Open Bottle, OP）、盖盖子（Close Cap, CP）等这一类多阶段、接触复杂的任务。
- **评价指标**: 任务成功率、操作过程中的平均接触力（验证Gentleness）。

**对比基准**:

- **DP (RGB)**: 仅使用RGB图像的Diffusion Policy。
- **DP3 (Visual PC)**: 仅使用视觉点云的3D Diffusion Policy。
- **人工示教 (Teleop)**: 人类遥操作采集的数据（用于对比数据质量和成本）。

## 5. 核心结果与关键Insight

**核心数据**:

- **样本效率**: 仅需50条高质量生成数据即可训练出高成功率的策略。
- **性能对比**: VT-DP在所有任务上均优于DP和DP3。特别是OP和CP任务，VT-DP成功率显著更高，证明了触觉引入的必要性。
- **数据质量**: 相比人工示教，VASK生成的数据轨迹更平滑，且通过RL强制约束了接触力，使得最终策略在执行时施加的力更小、更稳定（"Gentle"）。

**关键Insight**:

- **点云是视触觉融合的最佳桥梁**: 视觉和触觉在像素空间差异巨大，但在3D空间（点云）可以自然统一。将触觉传感器读数转换为局部物体表面点云，可以直接补充视觉点云在接触区域的盲区。
- **分层生成优于端到端规划**: VLM擅长高层逻辑，RL擅长底层控制。通过“VLM规划动作序列 -> RL执行原子技能”的方式，规避了VLM在底层控制上的幻觉问题，同时赋予了策略物理意识。

## 6. 待解决问题与未来方向

**局限性**:

- **Sim-to-Real Gap**: 尽管使用了数字孪生和点云对齐，仿真与真机的触觉信号（尤其是接触刚度和形变特性）仍存在差异，可能影响极其精细的操作。
- **泛化性**: 目前依赖预定义的原子技能库，对于技能库之外的复杂动作可能无法覆盖。

**未来方向**:

- **扩大技能库**: 引入更多样的原子技能以覆盖更广泛的任务。
- **在线适应**: 在真机部署时结合在线学习微调策略，以适应真实的物理特性。
