## 第一部分：Paper Research Agent —— 论文搜集这件事，以后不用人做了

你有没有试过，在Google Scholar上搜一篇论文，看了它的引用，又看了引用的引用，又看了相关工作推荐的论文，然后一个下午过去了，你还在找论文，还没开始读。

**这不是搜集，这是迷路。**

我们的Paper Research Agent做的是真正的**深度论文搜集**：

你只需要说一句话：
> "帮我找触觉条件扩散策略相关的论文"

Agent会自动：
1. **扩展搜索范围** — 从"触觉条件扩散"扩展到"触觉编码"、"扩散策略"、"触觉+视觉融合"等相关话题
2. **多轮迭代搜索** — 第一轮找到核心论文 → 从引用中发现更多 → 从相关工作中补充 → 三轮后覆盖面接近完整
3. **指定维度搜索** — 你可以指定话题、指定问题、指定方法，Agent会沿着这三个维度分别深入

**一次搜索 = 一个领域的完整文献综述**

而且这不是普通的"搜个名字"式搜集。Agent会读取每篇论文的全文，理解它在讲什么，然后按照我们制定的标准格式存入仓库。

## 第二部分：标准化论文仓库 —— 我们定下了AI论文库的标准

在做这个项目的过程中，我们发现一个问题：**每个人的论文都是乱放的。** 文件名是默认的，分析是写在Word里的，引用关系是记在脑子里的。换个电脑就全没了，换个合作者就说不清楚了。

所以我们制定了一套**标准化的论文仓库体系**：

| 标准 | 内容 |
|------|------|
| **PDF命名** | `{论文名}_{arXivID}.pdf`，一眼识别，不会重复 |
| **AI解析格式** | 每篇论文提取：问题、方法、实验、核心贡献、引用关系、验证状态 |
| **分析文档** | Markdown格式的深度分析，AI能读，人也能读 |
| **数据结构** | JSON Schema定义的论文-问题-方法三元组 |

**这套标准的价值是什么？**

任何人、任何团队、任何领域，都可以直接用这套体系来建自己的论文仓库。你不需要从零开始想怎么组织论文，我们已经帮你定义好了。

**无限扩展，无限加论文，永远不会乱。**

我们的仓库现在已经按照这套标准收录了74篇论文和52篇深度分析文档。

## 第三部分：前端可视化 —— 把顶级科学家的思维画出来

### 这是整个项目最核心的部分。

我们做了一件**前人从未做过的事情**：我们定义了顶级科学家脑内知识图谱的标准——**问题层级树**和**方法关系网**。

一个顶级教授看了100篇论文之后，他的脑子里是什么样的？

不是一堆论文标题，不是一张引用图。而是一个结构：

> "这个领域有3个核心问题。问题A在2016年被提出，经历了3次方法迭代，目前还没有被完美解决。问题B和问题C有交叉，但目前没有人同时处理它们。方法X是目前最有潜力的，但它在泛化性上有瓶颈。方法Y和方法Z的组合——没有人试过。"

**这就是顶级科学家的思维。这不是记忆力好，这是结构化思考。**

以前这种思维只有教授能做到，因为他们花了十几年训练。现在我们用AI和可视化，把这种思维变成了每个人都能用的工具。

### 我们和现有工具的本质区别

**Connected Papers、Semantic Scholar、Google Scholar——它们都是通过引用关系把论文连起来的。**

引用关系是什么？A论文引用了B论文，所以A和B有关系。这很表面。A可能引用了B只是为了说"我们的方法和B不同"，它们解决的可能完全是不同的问题。

**我们做的是语义理解。** 我们真正读懂了每篇论文在说什么：

- 它试图解决什么**具体问题**？
- 它提出了什么**方法**？
- 这个方法**验证了吗**？在真实机器人上还是仿真里？
- 它和**其他论文**的关系是什么？是同一个问题的不同解法，还是不同问题的交叉？

**他们连接论文。我们读懂领域。**

读懂了这个领域有哪些问题没被解决。读懂了哪些方法组合还没有人尝试。读懂了哪些方向最有价值但竞争最少。

### 七个视图

| 视图 | 它能帮你做什么 |
|------|--------------|
| **Problem Tree** | 一眼看清这个领域的问题是怎么分层的，从总目标到具体技术问题 |
| **Time Evolution** | 每个问题是哪年提出的，现在解决到什么程度了 |
| **Method → Problem** | 每个方法解决了哪个问题，箭头指向一目了然 |
| **Method Tree** | 方法之间的继承关系，PointNet怎么一步步演进到DP3 |
| **Dual Tree Fusion** | 问题树和方法树放在一起，交叉点就是还没人做过的创新机会 |
| **Paper Timeline** | 74篇论文按类别和时间分布，看清领域的发展节奏 |
| **Citation Network** | 1058条引用关系构成的论文网络，找出核心论文和桥接论文 |

**这不是图表。这是科研人员的决策工具。**

## 我们帮科研人员解决的三个问题

### 1. 找到真正有价值的问题

很多科研人员在做重复性的事情。他们选了一个问题，花了一年做出来，才发现别人去年已经做过了。

我们的Problem Tree标注每个问题的状态：solved / partial / active / unsolved。一眼就能看出哪些方向还有空间，哪些已经做烂了。

### 2. 洞察解决问题的最好方法

一个方法在什么条件下有效？它和哪些方法可以组合？它的瓶颈在哪里？

我们的Method Tree展示60个方法的完整继承链、跨域融合、目标映射。你不需要读完所有论文才知道方法之间的关系。

### 3. 发现真正的创新机会

**Dual Tree Fusion是我们最核心的视图。**

问题树和方法树放在一起。交叉点是什么？是"这个问题还没有用这个方法解决过"。那就是你的创新点。

**以前这是教授花几个月分析才能得出的结论。现在点一下就出来了。**
