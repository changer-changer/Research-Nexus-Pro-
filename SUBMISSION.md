## 痛点

一个研究生进入一个新领域，要做什么？

1. **花两周找论文** — 关键词换了20种，还是怕漏
2. **花一个月读论文** — 读完前面忘后面
3. **花两个月理关系** — 哪些论文解决同一个问题？哪些方法有继承关系？
4. **最后发现** — 自己选的问题别人已经做过了

**整个过程需要一个学期。我们把它压缩到了几分钟。**

## 我们做了什么

### 一、Paper Research Agent —— 深度论文搜集

**一句话触发，多轮深度搜索。**

不是搜个名字那种浅层搜集。你告诉Agent一个话题、一个问题、或一个方法，它会自动沿着三个维度展开搜索，多轮迭代，一次把相关论文全部找齐。

而且Agent会**读取每篇论文的全文**，理解它在讲什么，然后按照标准格式存入仓库。

### 二、标准化论文仓库 —— 我们定下了AI论文库的标准

我们制定了一套**标准化的论文存档体系**，让论文从命名到分析都有统一规范：

- **PDF命名标准**：`{论文名}_{arXivID}.pdf`，不重复，可识别
- **AI解析标准**：每篇论文提取问题、方法、实验、贡献、引用、验证状态
- **分析文档标准**：Markdown格式，AI能读，人也能读
- **数据结构标准**：JSON Schema定义的论文-问题-方法三元组

**这套标准是可复用的。** 任何团队、任何领域，拿过去就能用。你可以往里无限加论文，永远不会乱。

### 三、可视化知识图谱 —— 把顶级科学家的思维画出来

**这是前人没做过的事情。**

一个顶级教授看了100篇论文后，脑子里是什么样的？不是一堆论文标题，而是一个结构化的框架：

> "这个领域有3个核心问题。问题A经历了3次方法迭代，还没被完美解决。问题B和问题C有交叉，但没人同时处理过。方法X和方法Y的组合——没有人试过。"

**我们把这种思维框架可视化了。** 定义了一套标准——问题层级树和方法关系网，把教授脑内的分析过程变成了每个人都能用的交互工具。

#### 和现有工具的本质区别

**Connected Papers、Semantic Scholar、Google Scholar——它们通过引用关系把论文连起来。**

我们做的是**语义理解**。我们真正读懂每篇论文在说什么：它解决什么具体问题？提出什么方法？验证了吗？和什么其他工作有关？

**他们连接论文。我们读懂领域。**

读懂了哪些问题没被解决。读懂了哪些方法组合没人尝试。读懂了哪些方向最有价值但竞争最少。

#### 七个交互视图

| 视图 | 回答什么问题 |
|------|-------------|
| **Problem Tree** | 这个领域的问题是怎么分层的？ |
| **Time Evolution** | 每个问题是哪年提出的？解决到什么程度？ |
| **Method → Problem** | 每个方法解决了哪个问题？ |
| **Method Tree** | 方法之间是什么继承关系？ |
| **Dual Tree Fusion** | 问题和方法的交叉点在哪？（=创新机会） |
| **Paper Timeline** | 论文按类别和时间怎么分布？ |
| **Citation Network** | 论文之间的引用关系是什么？ |

## 这个系统怎么用？

**每个人拿到这套系统后，用它来构建自己领域的知识图谱：**

1. 用Paper Research Agent搜集你领域的论文
2. Agent自动按标准格式解析和存档
3. 系统自动提取问题、方法、关系
4. 前端自动可视化生成交互图谱

**74篇论文、30个问题、60个方法——这些是我们用这套系统在"触觉+视觉+机器人操作"领域跑出来的demo。** 你拿到之后，可以搜你的领域，构建你自己的数据集。每个人的结果都是不一样的、个性化的。

## 技术上怎么做的？

**数据管线：**
```
Paper Research Agent → 论文PDF → Agent深度阅读 → 结构化提取 → 标准化存储 → 前端可视化
     一句话触发        标准命名      读取全文        JSON Schema    仓库标准      7个视图
```

**前端：** React + TypeScript + 自定义SVG可视化引擎，7个联动视图，支持缩放、拖拽、搜索、书签、导出。
