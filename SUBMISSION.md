## 我们做了什么？

### 第一部分：Paper Research Agent —— 你的论文搜集专家

**一句话触发，多轮深度搜索，一次性找到一个话题下所有关键论文。**

不是普通的"搜个名字"式搜集。我们的Agent会：

- **指定话题搜索**：你说"触觉条件扩散策略"，它会自动扩展到相关的策略学习、触觉编码、扩散模型
- **指定问题搜索**：你说"灵巧操作中的滑动检测"，它会沿着这个问题的历史脉络一路追溯
- **指定方法搜索**：你说"PointNet系列在机器人操作中的应用"，它会找到所有使用和改进PointNet的工作
- **多轮迭代**：第一轮找到核心论文 → 从引用中发现更多论文 → 再从相关工作中补充 → 三轮后覆盖面接近完整

**一次搜索 = 一个领域的完整文献综述**

### 第二部分：标准化论文仓库 —— 我们定下了AI论文库的标准

我们不只是搜论文，我们建立了一套**标准化的论文存档体系**：

| 我们定义的标准 | 说明 |
|--------------|------|
| **标准PDF命名** | `{论文名}_{arXivID}.pdf` 格式，一眼识别 |
| **标准AI解析格式** | 每篇论文提取：问题、方法、实验数据、核心贡献、引用关系 |
| **标准分析文档** | 深度分析的Markdown文档，便于AI阅读也便于人类阅读 |
| **标准数据结构** | JSON Schema定义的论文-问题-方法关系图 |

**这套标准的价值**：任何人、任何团队、任何领域都可以直接使用这套体系来构建自己的论文仓库。**无限扩展，无限加论文。**

我们的仓库现在已经收录了74篇论文、52篇深度分析文档，全部按照这套标准格式组织。

### 第三部分：前端可视化 —— 把顶级科学家的思维画出来

**这是核心创新，也是前人从未做过的事情。**

我们定义了一套**顶级科学家脑内知识图谱的标准**——问题层级树和方法关系网。

一位顶级教授看了100篇论文之后，他的脑子里是什么样的？不是一堆论文标题，而是：

> "这个领域有3个核心问题，其中问题A在2016年被提出，经历了3次方法迭代，目前还没有被完美解决；问题B和问题C有交叉，但目前没有人同时处理它们；方法X是目前最有潜力的，但它在泛化性上有瓶颈。"

**这种思维，以前只有教授能做到。现在我们用AI+可视化把它变成了每个人都能用的工具。**

## 我们在帮科研人员做什么？

### 1. 帮他们找到真正有价值的问题

很多科研人员在做重复性的事情。他们选了一个问题，花了一年做出来，才发现别人去年已经做过了。或者这个问题根本就没有价值——已经被人用更好的方法解决了。

**我们的Problem Tree会标注每个问题的状态**：solved / partial / active / unsolved。一眼就能看出哪些问题还有空间。

### 2. 帮他们洞察解决问题的最好方法

一个方法在什么条件下有效？在什么条件下失效？它和哪些方法可以组合？它的瓶颈在哪里？

**我们的Method Tree回答这些问题。** 60个方法的继承链、跨域融合、目标映射，全部可视化。

### 3. 帮他们发现真正的创新点

**Dual Tree Fusion是我们最核心的视图。** 它把问题树和方法树放在一起，交叉点就是"这个问题还没有用这个方法解决过"——那就是创新机会。

## 技术上我们怎么做的？

### 数据管线

```
Paper Research Agent → 论文PDF → Agent深度阅读 → 结构化提取 → 标准化存储 → 前端可视化
     一句话触发        标准命名      读取全文        JSON Schema    仓库标准      7个视图
```

### 标准化体系

- **PDF命名标准**：`{论文名}_{arXivID}.pdf`
- **AI解析标准**：每篇论文提取6类信息（问题、方法、实验、贡献、引用、评价）
- **数据存储标准**：JSON Schema定义的三元组（论文-问题-方法）
- **分析文档标准**：Markdown格式的深度分析报告

### 前端技术

- React 18 + TypeScript + Zustand
- 自定义SVG可视化引擎（支持缩放、拖拽、搜索、联动）
- 7个联动视图，一键切换
- 本地运行，无需服务器，无需注册
